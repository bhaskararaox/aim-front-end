Frequently Asked Questions

AIM-PACT or Architecture Insights Model - Platform Analysis Center tool is a very powerful, user friendly Trace data search using Big Data and Machine learning.
User can make custom queries using the search box build query. Access to the tool is restricted and can be requested through AGS.
  • Go to AGS
  • Select the access "BI Big Data - Hadoop - End Users - Production - Architectural Insights Model - PACT Read only".
  • Click on "Step 3: Provide Justification".
  • Add the required justification and click "Save".
  • Click "Send".
Once your request is approved, you will receive an email and you are ready to use AIM-PACT.
The very first time you access AIM-PACT tool, you will need to accept the terms for a Privacy & Legal notice. You must already have the regular user permission to use the tool (instructions in the previous question).
Administrator access allows searching by non-published information and can be requested through AGS.
  • Go to AGS
  • Search and select the access "AIM-PACT - Production - Admin User".
  • Click on "Step 3: Provide Justification", add the required justification and click "Save".
  • Click "Send".

Trace quality classification is a quality standard used to differentiate trace sets as follows:
  • Gold
    • Trace points selected based on workload characterization analysis
    • Fully validated - Functional(x86) and Performance (keiko/asim)
    • Correlated to Silicon measurements
    • Correlated Trace weights
    • Good for performance projections and architecture studies
  • Silver
    • Trace points selected based on workload characterization analysis
    • Fully validated - Functional(x86) and Performance (keiko/asim)
    • Not correlated to Silicon measurements
    • Trace Weights maybe present but not correlated to the Si measurements
    • Good for architecture studies
  • Bronze
    • Quick path for trace generation (Random trace offset selection)
    • Limited validation with functional and performance simulators
    • Not correlated to Silicon measurements
    • Trace weights not generated
    • Custom tracing requests (e.g. support future ISA)
 For every trace, disasm file is generated using functional simulator X86. Each opcode in disasm is decoded using the standard intel tool XED (X86 Encoder Decoder) using the following command:
xed -64 -d $opcode
For more information regarding XED refer to following link:
http://mjc.intel.com/xeddoc/ref-manual/html/index.html We are using the same list to identify segments out of tags currently.
Here is the list: Server, Client, Big Data, Artificial Intelligence, Cloud, Graphics, Media, Browser, Tablet, Memory. Data gets refreshed daily twice (7am & 4pm). Yes. As an added advantage, command line is supplied to perform basic queries directly from console using a python script.
